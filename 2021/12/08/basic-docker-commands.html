Basic Docker commands I can never seem to recall...

08 Dec 2021 in Programming Notes Tags:

  • Pull an image or repo
    docker pull <name>

  • List all images
    docker images

  • List containers (-a for listing stopped containers too)
    docker ps -a

  • Run/start/stop a container process
    (-d for detached mode, -p for binding ports, –network for connecting to network, –name for setting container name) docker run -d -p<host_port>:<container_port> --network <network_name> --name <some_container_name> <image:version>
    docker start <container>
    docker stop <container>

  • Start/stop with Compose
    docker-compose -f <filename>.yaml up
    docker-compose -f <filename>.yaml down

  • Get container logs (-f/–follow to follow output like tail)
    docker logs -f <container>

  • Run bash shell in container
    docker exec -it <container> bash

  • Remove all containers/volumes/images
    docker rm -f $(docker ps -a -q)
    docker volume rm $(docker volume ls -q)
    docker rmi -f $(docker images -a -q

  • List, create networks
    (containers run with docker-compose are grouped to default network) docker network ls
    docker network create <some_network_name>
